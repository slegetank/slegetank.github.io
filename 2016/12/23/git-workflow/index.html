概述

git在使用流程上与svn最大的差别大概就是分支的使用了，本文也是围绕这个完成的。
在使用git时，所有的任务都是与分支相关的，所以一定要学会使用。

分支（一直存在的分支而非完成任务就删除的任务分支）至少要有两个，分别是开发分支和发布分支。开发分支用于平时团队开发使用（以及nightlies供QA），发布分支用于发布上架之后作为其他客户定制的稳定版本。这里有两种选择，分别是：

  1. master作为开发分支，另开一个release分支
  2. master作为release分支，另开一个develop分支

我们使用2。如果遇到因权限问题无法自己建立远端的分支,请联系管理员。

命令行

pull更新本地master分支

git pull origin master

不要在master分支上做出任何直接的commit。
当执行pull，merge或者rebase之后，请测试一下。git可能不会做出任何关于冲突的提示，但这并不代表没有编译错误。请确保你的每次commit不包含编译错误。

checkout一个特性分支

使用自己要做的任务作为分支名称，例如要添加一个音源tidal，则可以使用下面的命令:

git checkout -b init_tidal

例如要修复一个jira上的bug”IOS-31”,则使用:

git checkout -b jira-ios-31

这些title可以让我们明确的知道这个分支所做的事情。

在自己的特性分支上工作，经常的提交

经常使用rebase来合并远端的更改

经常rebase来避免你的分支与远端差别太大。

git fetch origin master
git rebase origin/master

这等同于下面的步骤:

git checkout master
git pull
git checkout jira-ios-31
git rebase master

将你的更改merge进master分支

当完成了要做的任务之后，需要将更改merge进master分支。

git checkout master
git merge jira-ios-31

将你的修改push到远端

最后需要将master push到远端。

git push origin master
git branch -d jira-ios-31

SourceTree

SourceTree 对工作流有着良好的UI支持，它默认的配置是将master作为稳定分支，将develop作为开发分支。它将命令行中描述的繁琐的命令行很好的集成进了UI中。作为mac上最好用的git工具之一，推荐使用。
